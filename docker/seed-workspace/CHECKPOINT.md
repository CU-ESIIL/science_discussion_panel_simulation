# Checkpoint

Use this file during manual or agent-assisted checkpoints. A checkpoint should make current state visible and recoverable.

## Checkpoint Prompt

Summarize:

- What changed since the last checkpoint
- Which files were created or edited
- Which decisions should be recorded
- Which assumptions changed
- Which outputs need review
- Which questions should be batched for the user
- Whether secrets, private data, or sensitive claims are present
- Whether a commit, issue, pull request, or release is appropriate
