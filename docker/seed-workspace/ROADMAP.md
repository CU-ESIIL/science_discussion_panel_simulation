# ROADMAP.md - Environmental Data Research Roadmap

| Phase | Purpose | Lead roles | Expected artifacts | Exit criteria |
| --- | --- | --- | --- | --- |
| Phase 0: Project charter | Define the project scope, boundaries, collaborators, and review requirements. | Scientific Director; Deputy Director / Integrator | Charter draft in `documents/`; initial `DECISIONS.md` and `ASSUMPTIONS.md` entries | Human lead agrees the scope is bounded and review requirements are explicit. |
| Phase 1: Research question and theory of change | Convert broad interest into answerable questions and a theory of how evidence informs action. | Scientific Director; Domain Scientist; Societal Impact / Translation Agent | Research question memo; theory-of-change sketch; audience map | Research questions are specific enough to guide data inventory and analysis. |
| Phase 2: Data inventory and provenance | Identify candidate datasets, access methods, licenses, limitations, and citation needs. | Data Engineer / Infrastructure Scientist; Citation & Evidence Curator; Domain Scientist | Data inventory; provenance table; license notes | Data sources have documented access paths, constraints, and citation requirements. |
| Phase 3: Reproducible analysis scaffold | Establish scripts, folders, environment notes, and conventions for repeatable work. | Data Engineer / Infrastructure Scientist; Quantitative Modeler; Technical Communicator | Script scaffold; analysis README; environment notes | A reviewer can understand how analyses will be run and where outputs belong. |
| Phase 4: Exploratory analysis | Explore patterns, anomalies, missingness, spatial/temporal coverage, and initial relationships. | Quantitative Modeler; Domain Scientist; Data Engineer / Infrastructure Scientist | EDA notes; preliminary tables; exploratory figures | Findings are labeled exploratory and key data issues are documented. |
| Phase 5: Modeling and uncertainty | Run models or quantitative syntheses with diagnostics, uncertainty, and sensitivity checks. | Quantitative Modeler; Domain Scientist; Skeptic / Adversarial Reviewer | Model scripts; diagnostics; uncertainty memo | Model assumptions, diagnostics, and limitations are explicit. |
| Phase 6: Figures and interpretation | Create figures and interpret results with provenance and caveats. | Quantitative Modeler; Domain Scientist; Scientific Narrative Lead | Figure files; figure provenance notes; interpretation memo | Each figure links to a script/data source and has reviewed caveats. |
| Phase 7: Skeptic review | Stress-test claims, assumptions, data provenance, and alternative explanations. | Skeptic / Adversarial Reviewer; Citation & Evidence Curator; Scientific Director | Skeptic review memo; revisions list; unresolved objections | Major objections are resolved, accepted as limitations, or escalated to human review. |
| Phase 8: Writing and citation audit | Draft the scientific narrative and audit every major claim against evidence. | Scientific Narrative Lead; Citation & Evidence Curator; Technical Communicator | Draft report/manuscript; citation audit; claim-evidence map | Claims are traceable, caveated, and approved for human review. |
| Phase 9: Impact translation | Translate reviewed findings into responsible audience-specific language. | Societal Impact / Translation Agent; Scientific Director; Domain Scientist | Impact memo; audience notes; misuse/risk notes | Translation stays within reviewed evidence and sensitive claims are flagged. |
| Phase 10: Release package | Assemble reproducible outputs, documentation, review notes, and release checklist. | Deputy Director / Integrator; Technical Communicator; Scientific Director | Release README; final artifacts; review checklist | Human lead approves publication, sharing, or archiving. |
