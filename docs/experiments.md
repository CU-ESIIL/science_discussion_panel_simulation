# Experiments

Experiments are small, bounded, and traceable by default. They clarify
disagreements; they are not automatically definitive science.

Each experiment should record title, question, hypothesis, proposer, reviewer,
data source, code path, environment, compute estimate, approval status, stopping
rule, result, uncertainty, limitations, discussion impact, and reproducibility
command.

Use existing local scripts or bounded worker mechanisms. Do not launch
expensive, destructive, billed, or long-running jobs without approval.
