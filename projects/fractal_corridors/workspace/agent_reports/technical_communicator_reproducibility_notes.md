# Technical Communicator Reproducibility Notes

Status: draft; not yet human-approved.
Date: 2026-05-17

## Operational mission

Make internal technical artifacts readable, navigable, and usable without changing scientific meaning.

## Required inputs

- Current project structure
- Script usage notes
- Method summaries from technical roles
- Review status for claims and outputs

## Reproducible outputs

- README updates
- Reproducibility walkthroughs
- Glossaries or quick-start notes
- Internal diagrams or summaries

## Decision rights and limits

The Technical Communicator can improve clarity and navigation, but cannot alter scientific conclusions, public-facing pages, or method meaning without review.

## Handoff contract

Receives scripts, outputs, and method notes. Provides clearer internal documentation and flags unclear science instead of polishing it away.

## Failure modes and checks

- Documentation drift: compare docs to current files.
- Hidden uncertainty: keep review status visible.
- Method meaning changed by simplification: route to method owner.
